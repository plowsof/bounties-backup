Implement viewkey-based background refresh to significantly reduce wallet sync time

Posted by
cryptogrampy@protonmail.com
10 months ago
Description

It's been discussed that one could utilize view keys to securely allow wallets (Monero GUI, mobile wallets, etc) to continuously background sync without having to keep the wallet open, hot and unlocked.

From a user's perspective, one would be able to leave their mobile wallet alone for a week, open the wallet up, and their wallet will already fully synced (given they are continuously connected to the internet/wifi, power, etc).

HYC wrote up a Monero issue describing a rough idea of what needs to be done to make this happen: https://github.com/monero-project/monero/issues/8082

Additionally,

There are 2 other existing Monero bounties that are essentially this same idea (one targeted at Monerujo wallet, and one for Cake), but THIS bounty is for adding the feature into the Monero libwallet core codebase first- the other bounties would be simply implementing this newly added cross-wallet Monero core enhancement on their front ends. The existing mobile wallet bounties are separate from this bounty.

Requirements:

Code is open-source
Code is submitted to the Monero repo as a pull-request (https://github.com/monero-project/monero/pulls)
Work with Monero devs and mobile wallet devs (Monerujo, Cake, etc) to determine agreed-upon solution
Pull request properly improves wallet sync time
Pull request is accepted into Monero's code base after review
