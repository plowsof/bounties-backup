I think this is a fantastic idea, and badly needed in the space.

This bounty is a bit light on technical detail, and I've put a fair amount of thought into the subject, so I'll add some potential requirements and concerns.

Concerns:

There is a significant danger in creating something like this as you're going to be embedding a spend key in the QR, and the person redeeming it will not know where to redeem, or the website provided on the card may be defunct or not trustworthy, the creator of this bounty may decide to rugpull and just start stealing gift card money, etc etc etc. NOT COOL.
Or perhaps redeemers will go to a search engine, and some scam website 'RedeemMoneroGiftCard.com' will pop up, and the person redeeming will again lose all their money.... The most difficult/security problematic part of this bounty is probably the redeeming side. Imagine someone who doesn't know anything about this stuff trying to use this thing. I don't think they should have to download an app that is made solely for this gift card bounty, either... it doesn't seem like it would be a good experience.

I was going to put this as a 'suggested' feature, but frankly, a community approved wallet (Monerujo/Cake/Monero GUI) should really add the ability to redeem wallets using the Monero Wallet redeem URI (https://github.com/monero-project/monero/wiki/URI-Formatting.) prior to any work being done on this bounty. I'm also not aware of any wallets that actually implement it currently. That's my personal opinion, but happy to hear discussion on this.

But. Perhaps this bounty should solely geared just towards gift card creation? I think I would be cool with that. In which case:

Requirements (geared toward creation of gift cards... again, i don't think this bounty should involve the redeeming side):

  • Should use an existing and community approved Monero library for generating seed/wallet
  • Seed should have secure entropy/randomness
  • Users should be able to fund the gift card by scanning a QR code (perhaps it's a few step process- 1. create wallet, 2. fund wallet 3. print)
  • Users should be able to see how much they've added to the card after sending funds.
  • Should follow the Monero URI protocol: https://github.com/monero-project/monero/wiki/URI-Formatting. The QR code for the wallet to be redeemed should probably be created from this exact URI.
  • Should provide a gift card for user to print
  • Should have printing ability
  • Should be open source code and freely available for anyone to use as they wish
  • Be able to use your own node and remote node for creating
  • The generator app must be self-hostable/fully client side code if web app. It should 100% not rely on an opaque backend that generates seeds, etc.
  • The gift card output should have simple instructions for how to redeem and a link to suggested wallets that can be used for redeeming.

I want to lay down my thoughts on this bounty:

  • I want the card be the size of a credit card - it fits right in the wallet along with the rest of the cards and can be stashed in a pocket.
  • I think the quality of the card should match that of a regular business card. I don't think it's possible to make one in home conditions hence I would be okay with a) paying for professional service to print them for me; b) purchasing them from an eshop.
  • Because I can't trust printing services or an eshop with my QR codes, I think the card should be sold blank, without a QR code. Printing QR codes at home is less demanding and can be done on any old school ink printer. After that, the QR code is glued onto the card. Design must be explicit where on the card place the QR code.
