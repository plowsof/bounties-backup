This bounty was discussed in the Monero UX channel on 11/11/21 via a collaborative Figma whiteboarding session. The mockup includes requirements from the bounty (as well as future goals) and is a suggested direction / design to consider when building this app. Figma link: https://www.figma.com/file/Xt0J6n8HOg26y9DiyL7CXK/Monero-Node-GUI-Service?node-id=0%3A1

cryptogrampy@protonmail.com
11 months ago

Just want to say that this one is in progress and just about completed- should be done this week. It will have start/stop/autostart functionality (user would need to autostart the app itself on boot), and it will have the ability to input a config and specify a file path to monerod and it will display data coming from Monerod.

I will not be including upgrading functionality as it's a bit more controversial / dangerous (in my opinion), but I will make it so the UI tells the user that an update is available. I will be releasing the code as a Hello World Monero Desktop App and will provide a demo of it in action but will likely not be releasing any binaries as that is also a significant amount more work.

Beyond this, I will be creating another related to create something that I think more users in the Monero ecosystem will be able to contribute to, and to take this app further.
