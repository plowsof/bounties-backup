Android GUI Miner App for running XMRig, P2Pool, etc.

2 months ago cryptogrampy@protonmail.com
Description

This is a request to create a GUI Android application that allows users to easily run XMRig and mine on their Android phones.

While there are a few apps out there of varying quality and up-to-dateness with regard to XMRig, there aren't any apps that allow
users to easily provide their on XMRig config, and don't take any percentage off the top.

Minimum Requirements:

  • Provide an open-source, completely user buildable Android APK written
  • Should allow users to Start, Stop, and set # CPU cores for mining
  • Uses the latest XMRig and up-to-date GUI language (Android, Java, Flutter, etc)
  • Runs on ARMv8
  • Freely available for anyone to fork etc
  • Does not include a hard-coded donation
  • Is free to download in all stores
  • Can functionally work when pointed at various pools: MoneroOcean and P2Pool as mandatory examples
  • Displays current hashrate

Nice to Haves:

  • Extensible code that people can easily add features to.
  • Ability to set to run whenever plugged in
  • Ability to set mining schedule (e.g. between 6 and 12 pm)
  • Ability to turn off if battery hits certain percentage
  • Adequate logging
  • Available on FDroid
  • Available on Play Store
  • Live charts of hashrate (as seen in React Native XMRig as well a others)

Payout Criteria:

  • Code is open-sourced
  • Demo of the miner
  • Miner can be built / tested by community member

Im thinking how to design the settings. We want to allow advance users to past an xmrig config and I think we have to allow a simple mode too

  1. Do we ask the user to select advance mode or simple mode? Because enable both of them can cause a race condition - like which pool to use, the one you entered in simple more or the one you pasted in config editor.
  2. What we what to allow in simple mode. Pool, wallet, cpu hint.. what more?
  3. Do we want to enable/disable algos in simple mode?
  4. Do we want to use moneroocean xmrig version or the original? MO version is better but its have some issues in benchmark (mandatory) is some devices (process stuck on some algos).

In mining screen you can choose the configuration to use.

I think this is the best approach in terms of UI/UX.
