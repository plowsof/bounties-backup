Monero tip bot for Mastodon/Pleroma

Summary

  • Monero tip bot plugin for Mastodon/Pleroma.
  • A KYC-free Monero tip bot could give people an incentive to move from Twitter (centralized, censored, Tor-hostile, KYC'ed tips) to Mastodon or Pleroma (open source, federated, privacy-friendly, censorship-resistant).
  • Donors tip to support content creators (e.g. tip a text post, article, image or video).
  • Right now, people can add their Monero address to their bio or pinned post, but a "Tip" button (similar to Noise.cash) would make it more convenient to tip.

Basic Functionality

  • Server admins can install the Tip plugin to their server.
  • For every post, a "Tip" button is displayed next to the "Repost" and "Like" buttons.
  • Next to the Tip button, the total donation amount is displayed. Total donation amounts are per post, not per user.
  • The Tip plugin supports external wallets (donors can send Monero with their external wallet and content creators can receive Monero to their external wallet). Maybe an optional web wallet feature could be included.

Steps

  1. Content creators create a web wallet or enter their external Monero address (e.g. from the GUI Wallet or Cake Wallet).
  2. Donors click the "Tip" button, enter a USD or Monero amount, and pay it with their web wallet or via an external wallet (the address, amount and QR code are displayed).
  3. The tip transaction is detected and the total donation amount is updated. (Via view key or an intermediate custodial wallet.)
  4. The tip is forwarded from the intermediate wallet to the content creator (e.g. after reaching the minimum withdrawal amount to avoid dust TXs) or the donor sends the tip directly to the content creator (no intermediates).

UI Idea

Here is a basic UI design, but feel free to use something else: https://ibb.co/m6xpvRr

Info

As far as I know, Mastodon and Pleroma are interoperable. However, if you need to decide between supporting Mastodon or Pleroma first, I would recommend Pleroma because it's more lightweight, supports longer posts (up to 5000 characters) and has a chat feature.

It seems like Mastodon and Pleroma don't have a simple way to support add-ons yet.

A simple tip bot that is called in a comment and tips the OP may be easier for now.

The Mastodon/Pleroma comment tip bot could integrate Monero RPC (possibly with a remote node, so you don't need an expensive storage VPS) & create a separate account per user who enables tip bot, allow external deposits to your tipping balance, and allow withdrawals to external addresses.

Both Mastodon/Pleroma support private messages. Pleroma additionally has a chat feature.

There is already a Reddit tip bot that could be ported:
Mastodon API (Python): https://github.com/halcy/Mastodon.py
Reddit tip bot (Python): https://github.com/monero-ecosystem/MoneroTipper

This would be centralized and custodial sadly (although multiple people can run alternative tip bots), but it would probably need to work like this if it's a comment bot, compared to an integrated feature.

Another idea is Monero tip support for Lemmy (federated open source alternative to Reddit), but I'm more interested in Mastodon/Pleroma right now.
