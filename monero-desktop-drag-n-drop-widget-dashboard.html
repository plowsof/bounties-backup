Bounty Intro:

While working on another bounty: https://bounties.monero.social/posts/8/desktop-service-gui-for-running-a-monero-node-on-boot-in-the-background, I thought of a potentially very cool idea... rather than creating a one off desktop app, why not build out a desktop drag n' drop widget dashboard that:

  1. Allows users to create their own custom dashboards
  2. Allows Monero developers to create and publish awesome monero experiences without having to constantly re-invent the wheel.

Drag and drop examples (image search for 'drag and drop dashboard) :
https://gridstackjs.com/demo/react.html

https://kalitte.gallerycdn.vsassets.io/extensions/kalitte/jdash-aspnetmvcdashboardframework/1.0/1482140221164/134416/1/ipad.png

So for the other bounty I'm currently working on, if a user wanted to use a timer to control when Monerod turns on and off, they could drag a Monerod Controller widget onto their dashboard, add the monerod file location and config in the widget, and then drag the Monerod Timer widget off of the sidebar, and specify their monero start/stop times (when they want Monerod to run).

Other widget ideas:

  • Monerod data chart that displays live monerod data in a chart format
  • Monero price converter
  • Monero mining controller
  • Monero Updater
  • Monero CCS display (similar to Feather Wallet)

Obviously, a person could go pretty hog wild on this stuff and it could even tap into other things (i2p/tor, xmrig, monero-wallet, plowsof's funding as a service, etc...

BOUNTY REQUIREMENT:

The initial goal of this bounty (for payout) is:

  • create the proof of concept in Electron using a popular front end framework and drag and drop library. It will be rough and not perfect, and mostly meant to show the idea.
  • create a drag and drop Monerod controller widget that can connect to Monerod
  • create a drag and drop timer widget that can specify monerod start and stop times.
  • create a data display widget that displays current monerod data
  • the dashboard is persistent and will retain the widgets upon restart
  • Code is all open source FOREVER, forkable, etc etc.

stretch goals (not required):

  • the widgets do not start running until placed on the dashboard and stop when they are removed from the dashboard
  • the widgets can be resized
